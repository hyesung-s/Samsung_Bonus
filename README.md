# 2026 삼성전자 성과급 계산기

사용자가 입력한 가정값을 기준으로 OPI1/OPI2 성과급을 계산하는 정적 웹앱입니다. GitHub Pages에 그대로 올려 사용할 수 있습니다.

## 주요 기능

- 메모리, 파운드리, S.LSI, 공통 4개 조직 반영
- OPI1 = 본인 연봉 × OPI1 비율
- OPI2 부문 재원은 사업부 전체 예상 영업이익 합계에 적자를 그대로 반영
- OPI2 사업부 재원은 사업부별 예상 영업이익이 흑자인 경우에만 반영
- 공통 조직은 선택한 사업부 재원에 `공통 인원 × 공통 가중치`로 반영하는 재원보존형 계산
- 본인 연봉, 사업부, 평균연봉, 사업부별 영업이익, 인원, 공통 가중치 수정 가능
- OPI 조건은 접이식 고급 설정으로 제공
- 사업부별 OPI1, OPI2 부문, OPI2 사업부, OPI2 합계, 총 성과급 시각 비교
- 현재 설정 링크 복사 기능
- 기본값 복원 기능
- 입력값 자동 저장
- 모바일/PC 반응형 레이아웃

## 계산식

```text
OPI1 = 본인 연봉 × OPI1 비율

OPI2 부문 재원
= SUM(사업부별 예상 영업이익) × OPI2 재원 비율 × 부문 배분 비율

OPI2 사업부 재원
= MAX(사업부별 예상 영업이익, 0) × OPI2 재원 비율 × 사업부 배분 비율

부문 가중인원
= 메모리 인원 + 파운드리 인원 + S.LSI 인원 + 공통 인원 × 공통 가중치

일반 사업부 OPI2 부문
= OPI2 부문 재원 ÷ 부문 가중인원

공통 OPI2 부문
= 일반 사업부 OPI2 부문 × 공통 가중치

선택 사업부의 사업부 가중인원
= 선택 사업부 인원 + 공통 인원 × 공통 가중치

공통 OPI2 사업부
= 선택 사업부의 OPI2 사업부 기준금액 × 공통 가중치

본인 OPI2
= 평균연봉 기준 OPI2 × 본인 연봉 ÷ 평균연봉
```

## GitHub Pages 배포 방법

1. GitHub에서 새 repository를 만듭니다.
2. 이 폴더의 파일을 repository 루트에 업로드합니다.
   - `index.html`
   - `styles.css`
   - `app.js`
   - `README.md`
3. repository의 **Settings → Pages**로 이동합니다.
4. Source를 **Deploy from a branch**로 선택합니다.
5. Branch는 `main`, folder는 `/root`로 선택합니다.
6. 저장하면 GitHub Pages 주소가 생성됩니다.

## 주의사항

이 웹앱은 사용자가 입력한 가정값으로 계산하는 비공식 시뮬레이터입니다. 실제 회사 제도, 세금, 개인별 평가, 지급 대상, 지급 시점, 지급 상한 등과 다를 수 있습니다.
